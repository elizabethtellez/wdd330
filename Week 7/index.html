Chapter 13 - AJAX

  • Fetch requests are so incredibly important for 90% of real web applications. A lot of the apps we build at the school are just toys,
    AJAX is the gateway to larger more complex apps.
  • It's interesting that the book seperates the headers and request objects into their own variables. I usually just use the 'fetch'
    function's interface to define that stuff. If I even need too, I usually find the default headers acceptable.
  • Please don't import jQuery just to make requests.
  • I like that general 'curry' function, but I cant see much use for it.
  • There are 7 steps to on how AJAX works
    1. An event occurs in a web page (the page is loaded, a button is clicked)
    2. An XMLHttpRequest object is created by JavaScript
    3. The XMLHttpRequest object sends a request to a web server
    4. The server processes the request
    5. The server sends a response back to the web page
    6. The response is read by JavaScript
    7. Proper action (like page update) is performed by JavaScript
