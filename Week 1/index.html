Reading: Doing Stuff with Web Things

  • Something that caught my attention is the difference between Cookies and Local Storage.
  • Cookies: are send to the server in e ach HTTP header
  • Information stored or shared – output
  • Anything coming from where it was stored or anything user enters – input
  • Input-Output – IO
  • Section tags – fills up entire width of the rowli
  • Local storage – storage within html that can be looked at even after the web page refreshes - uses key-value pairs. Like a phone book, key = name of person, value = phone number

MOBILE UX notes from sample

  • It is interesting how the web has shifted from being mostly desktop dominated to now being mostly viewed through mobile devices!
  • They make a good point that we should be designing for mobile first, since that's where our content is most likely going to be viewed.
  • Consolidate your user interfaces. Make common interactions as simple and easy as possible! Fewer steps are better!
  • Move important UI elements to the bottom of the screen, that is the spot that is easiest to reach.
  • Start with the smallest screens in mind
  • Desktops are usually shared in the home
  • Mobile devices turnover is very high, every 2 years or so
  • Start small and scale up
