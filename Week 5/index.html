Week 5 Notes

Chapter 10 - Testing and Debugging

  • Errors – caused when something goes wrong in a program
      • System error – system or external devices and interaction with program
      Programmer error – incorrect syntax or faulty logic; could be simple as a typo
      User error – entered data incorrectly, program was unable to handle
  • Exception – return error that can be used by program to deal with error
  • Stack traces – sequence of functions or method calls that lead to point where error occurred
  • Warning – If there’s error in code that isn’t enough to cause program to crash
  • ‘use strict’; - considers poor styles as actual errors
  • Alert() – show dialog at certain points of code
  • Try – block that can catch exception
  • Catch – block that receives try block error
  • Finally – block that will always execute
  • Create error objects with: Const error = new Error(“Error Message”);
  • Throw error with: throw “message” or throw new Error(message).
